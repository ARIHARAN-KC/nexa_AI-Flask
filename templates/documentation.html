Getting Started

Installation

Nexa AI is available as a web application and also offers a CLI tool for local development. To get started with the web version, simply create an account at our registration page.

For CLI installation:

# Install Nexa CLI with npm
npm install -g nexa-ai-cli

# Authenticate with your account
nexa login

# Verify installation
nexa --version

Your First Project

Creating your first project with Nexa AI is simple. Follow these steps:

  1. Navigate to your workspace after logging in
  2. Click "New Project"
  3. Describe what you want to build in natural language
  4. Select your preferred tech stack (or let Nexa AI recommend one)
  5. Click "Generate Project"

Tip: Be as specific as possible in your project description. Include details about features, target users, and any technical requirements.

Features Guide

Project Generation

Nexa AI can generate complete projects from high-level descriptions. Here's an example of an effective prompt:

# Example project description
"Create a task management web application with the following features:
- User authentication (signup/login)
- CRUD operations for tasks
- Task categories and tags
- Due date reminders
- Responsive design

Use React for the frontend and Node.js with Express for the backend.
Database should be MongoDB.
Include unit tests and API documentation."

Nexa AI will generate:

  • Complete project structure
  • All necessary configuration files
  • Boilerplate code for all components
  • Database models and API routes
  • Basic styling
  • Test files
  • Documentation

Code Assistance

While working on your project, you can get AI assistance for:

Code Completion

Get intelligent suggestions as you type. Press Tab to accept suggestions.

Documentation

Generate docstrings and documentation by typing /// above a function.

Refactoring

Select code and choose "Refactor" from the context menu for optimization suggestions.

Code Explanation

Highlight code and ask "Explain this" in the chat panel.

Debugging

Nexa AI provides several debugging tools:

Error Analysis

When you encounter an error:

  1. The AI will automatically analyze the error message
  2. It will suggest potential fixes
  3. You can ask follow-up questions in the chat
# Example error analysis
Error: Cannot read property 'map' of undefined
Solution: The data variable is undefined. Add a null check:
{data && data.map(item => (...))}

Pro Tip: For complex debugging sessions, use the "Debug Mode" which provides step-by-step analysis and keeps track of all suggested solutions.

API Reference

Authentication

All API requests require authentication using your API key. Include it in the Authorization header.

# Example authenticated request
curl -X GET \
-H "Authorization: Bearer YOUR_API_KEY" \
https://api.nexa-ai.com/v1/projects

You can find your API key in:

  • Web: Account Settings â†’ API Keys
  • CLI: Run nexa api-key

Projects API

Endpoint Method Description
/v1/projects GET List all projects
/v1/projects POST Create new project
/v1/projects/{id} GET Get project details
/v1/projects/{id}/generate POST Generate project code

Create Project Example

POST /v1/projects
# Request body
{
  "name": "Task Manager",
  "description": "A web app for managing tasks with user authentication",
  "tech_stack": ["react", "node", "mongodb"],
  "features": ["auth", "crud", "reminders"]
}

AI Models API

You can directly interact with different AI models through our API:

POST /v1/ai/generate
# Request body
{
  "prompt": "Create a React component for a login form",
  "model": "gemini-pro", // Options: gemini-pro, chatgpt, cohere, deepseek
  "temperature": 0.7, // Creativity (0-1)
  "max_tokens": 1000 // Maximum response length
}

Note: The AI Models API has rate limits based on your subscription plan. Check your dashboard for usage statistics.

Best Practices

Writing Effective Prompts

  • Be specific about requirements and constraints
  • Mention your preferred technologies
  • Include examples of similar projects
  • Break complex requests into smaller steps
  • Specify the output format you need

Iterative Development

  • Start with a minimal viable product
  • Generate and test small components
  • Use the feedback loop to refine
  • Version your AI-generated code
  • Combine AI output with manual code

Security Considerations

Do:

  • Review all generated code for vulnerabilities
  • Use environment variables for secrets
  • Implement proper input validation
  • Add your own authentication layer

Don't:

  • Use AI-generated auth code without review
  • Include secrets in your prompts
  • Deploy without security testing
  • Assume AI code is production-ready

Frequently Asked Questions

How accurate is the generated code?

Nexa AI generates functional code in most cases, especially for common patterns and frameworks. However, you should always review and test the code, as the AI may make mistakes or use outdated practices in some cases.

Can I use Nexa AI for commercial projects?

Yes, you retain full ownership of all code generated by Nexa AI. However, we recommend reviewing our Terms of Service for specific details about commercial use.

What programming languages are supported?

Nexa AI supports all major programming languages including JavaScript/TypeScript, Python, Java, C#, Go, Rust, and more. Web frameworks (React, Vue, Angular), mobile (Flutter, React Native), and backend technologies are all well-supported.

How does pricing work?

We offer a free tier with limited generations per month. Paid plans provide more generations, access to premium AI models, and advanced features. See our Pricing page for details.

Can I train the AI on my own codebase?

Our Enterprise plan includes fine-tuning capabilities where you can train models on your private codebase to match your specific patterns and standards.
